
1. Platform Channels 개요

정의: 플러터 앱(Dart)과 호스트 OS(네이티브) 간에 메시지를 주고받을 수 있도록 연결해 주는 통신 통로입니다.

필요성: 카메라, 블루투스, 배터리 상태 등 OS의 특정 네이티브 API를 직접 제어해야 할 때 사용합니다.

2. 주요 채널 종류

MethodChannel: 가장 흔히 사용되며, 다트에서 네이티브 함수를 호출하고 결과값을 반환받는 1회성 통신에 적합합니다.

EventChannel: 센서 데이터나 네트워크 상태 변화처럼 네이티브에서 플러터로 지속적인 데이터를 보내는 스트림 통신에 적합합니다.

3. 통신 과정

Flutter: MethodChannel 객체를 생성하고 특정 이름(String)으로 네이티브 함수를 호출합니다.

Native: 각 플랫폼(Android/iOS)에서 같은 이름의 채널을 설정하고 호출을 감지하여 해당 로직을 실행합니다.

Result: 작업 결과를 다시 플러터(Dart) 쪽으로 응답합니다.

4. 핵심 요약
| 구분 | MethodChannel | EventChannel |
| :--- | :--- | :--- |
| 통신 방식 | 일회성 함수 호출 (Request-Response) | 지속적인 데이터 흐름 (Stream) |
| 주요 용도 | 기기 정보 조회, 특정 기능 실행 | 가속도계, GPS 위치 추적 |
| 데이터 변환 | 표준 메시지 코덱을 통해 자동 변환 | 표준 메시지 코덱을 통해 자동 변환 |
