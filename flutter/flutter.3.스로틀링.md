3. 스로틀링(Throttling)과 디바운싱(Debouncing)
두 기법 모두 짧은 시간 동안 발생하는 과도한 이벤트를 제어하기 위해 사용합니다.

스로틀링 (Throttling): 이벤트를 일정 시간 간격으로 최대 한 번만 실행합니다.

예시: 스크롤 이벤트. 사용자가 빠르게 스크롤할 때 0.1초마다 한 번씩만 특정 로직을 실행하여 성능 저하를 막습니다.

디바운싱 (Debouncing): 이벤트가 연이어 발생할 때는 대기하다가, 마지막 이벤트 발생 후 일정 시간이 지났을 때 비로소 실행합니다.

예시: 검색창 자동 완성. 사용자가 "Flutter"를 입력할 때 한 글자마다 API를 호출하지 않고, 입력을 멈춘 뒤 0.5초 후에 한 번만 서버에 요청을 보냅니다.
