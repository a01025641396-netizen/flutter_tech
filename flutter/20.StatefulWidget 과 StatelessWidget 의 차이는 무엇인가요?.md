1. StatelessWidget (상태가 없는 위젯)

정의: 한 번 생성되면 내부 데이터가 변하지 않는 정적인 위젯입니다.

특징: 위젯 자체가 **불변(Immutable)**이며, 화면을 다시 그리려면 부모 위젯이 새로운 데이터를 전달하며 위젯을 새로 생성해야 합니다.

예시: 로고 이미지, 단순 텍스트, 아이콘 등.

2. StatefulWidget (상태가 있는 위젯)

정의: 유저와의 상호작용이나 데이터 변화에 따라 화면이 동적으로 변하는 위젯입니다.

특징: 위젯 클래스와 상태를 관리하는 State 클래스로 나뉩니다. setState() 함수를 호출하여 UI를 실시간으로 갱신(Rebuild)할 수 있습니다.

예시: 체크박스, 입력창, 카운터 숫자, API 리스트 등.

3. 핵심 차이점 요약
| 구분 | StatelessWidget | StatefulWidget |
| :--- | :--- | :--- |
| 상태 변경 | 불가능 (정적) | 가능 (동적) |
| 화면 갱신 | 부모에 의해서만 가능 | setState()로 스스로 가능 |
| 생명주기 | 단순 (build만 존재) | 복잡 (initState, dispose 등) |
