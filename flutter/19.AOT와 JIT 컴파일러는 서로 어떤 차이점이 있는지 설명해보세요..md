1. JIT (Just-In-Time) 컴파일러
**"프로그램 실행 중에 실시간으로 코드를 기계어로 번역"**하는 방식입니다.

동작 방식: 소스 코드를 실행 시점에 필요한 부분만 그때그때 컴파일합니다.

주요 특징: 코드 수정 사항을 즉시 반영할 수 있는 유연성이 핵심입니다.

사용 시점 (Debug Mode): 개발 단계에서 **핫 리로드(Hot Reload)**를 가능하게 하여 개발 속도를 비약적으로 높여줍니다.

2. AOT (Ahead-Of-Time) 컴파일러
**"실행 전(배포 전)에 미리 소스 코드 전체를 기계어로 번역"**하는 방식입니다.

동작 방식: 앱을 빌드하는 시점에 타겟 기기(iOS, Android 등)의 CPU가 이해할 수 있는 바이너리 코드로 미리 변환해 둡니다.

주요 특징: 실행 시점에 컴파일 과정이 생략되므로 실행 속도가 매우 빠르고 안정적입니다.

사용 시점 (Release Mode): 실제 사용자에게 배포하는 단계에서 최고의 성능과 부드러운 UI 애니메이션을 보장하기 위해 사용합니다.

3. 핵심 차이점 요약
| 구분 | JIT (실시간 컴파일) | AOT (사전 컴파일) |
| :--- | :--- | :--- |
| 컴파일 시점 | 앱 실행 중 (Runtime) | 앱 빌드 시 (Build time) |
| 주요 장점 | 빠른 개발 속도 (Hot Reload) | 최적의 실행 성능 및 보안 |
| 적용 모드 | 디버그 모드 | 릴리스 모드 |
