21. 무한 스크롤, 당겨서 새로고침 사용 이유
1. 무한 스크롤 (Infinite Scroll)

사용 이유: 대량의 데이터를 한 번에 로드할 때 발생하는 메모리 과부하와 속도 저하를 방지하기 위함입니다.

특징: 화면 끝에 도달할 때만 데이터를 추가로 불러와 초기 실행 속도를 높이고 리소스를 효율적으로 관리합니다.

2. 당겨서 새로고침 (Pull-to-Refresh)

사용 이유: 사용자가 원할 때 즉각적으로 최신 데이터로 화면을 업데이트하기 위함입니다.

특징: 앱을 재시작하지 않고도 리스트를 갱신할 수 있는 직관적인 사용자 경험(UX)을 제공합니다.

3. 핵심 요약
| 구분 | 무한 스크롤 | 당겨서 새로고침 |
| :--- | :--- | :--- |
| 주요 목적 | 성능 최적화 (데이터 분할 로드) | 최신 데이터 동기화 |
| 사용자 경험 | 끊김 없는 콘텐츠 탐색 | 수동 데이터 갱신 |
| 주요 위젯 | ListView.builder, ScrollController | RefreshIndicator |
