5. RESTful API와 핵심 원칙
**REST(Representational State Transfer)**는 웹의 기존 인프라(HTTP)를 그대로 활용하기 위한 아키텍처 스타일입니다. 이를 따르는 API를 RESTful API라고 합니다.

6가지 핵심 원칙
Uniform Interface: 자원(Resource)은 URI로 식별하고, HTTP 표준 메서드(GET, POST, PUT, DELETE)를 통해 일관된 방식으로 조작해야 합니다.

Stateless (무상태성): 서버는 클라이언트의 문맥을 저장하지 않아 처리가 단순하고 확장성이 높아야 합니다.

Caching (캐시 가능): HTTP의 기존 캐싱 기능을 적용할 수 있어야 합니다.

Client-Server 구조: 클라이언트는 사용자 인터페이스를, 서버는 데이터 처리를 담당하여 역할을 명확히 분리합니다.

Hierarchical System (계층화): 클라이언트는 서버의 계층 구조(프록시, 게이트웨이 등)를 몰라도 통신이 가능해야 합니다.

Code on Demand (선택적): 서버에서 스크립트를 클라이언트로 보내 실행할 수 있습니다.
