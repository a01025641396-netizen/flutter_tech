3. HTTP의 주요 특징
1) 비연결성 (Connectionless)
내용: 클라이언트가 요청을 보내고 서버가 응답을 마치면 바로 연결을 끊어버리는 성질입니다.

장점: 서버가 연결을 계속 유지하지 않아도 되므로, 리소스를 아끼고 더 많은 클라이언트의 요청을 처리할 수 있습니다.

2) 무상태성 (Stateless)
내용: 서버가 클라이언트의 이전 상태(로그인 여부 등)를 기억하지 않습니다.

장점: 서버가 클라이언트의 상태를 관리할 필요가 없어 서버 확장이 매우 자유롭고 단순합니다.

3. HTTP의 문제점 (한계)
특징에서 파생되는 현실적인 문제점들이 있습니다.

1) 매번 새로운 연결 (비연결성의 한계)
문제: 요청을 보낼 때마다 TCP 연결(3-Way Handshake)을 새로 맺어야 해서 오버헤드가 발생합니다.

해결: 현대의 HTTP(1.1 이상)에서는 Keep-Alive 옵션을 통해 일정 시간 동안 연결을 재사용하며 이를 보완합니다.

2) 상태 기억 불가 (무상태성의 한계)
문제: 쇼핑몰 장바구니나 로그인 정보처럼 "누가 보낸 요청인지" 알아야 하는 상황에서 서버가 기억을 못 합니다.

해결: 쿠키(Cookie), 세션(Session), 또는 토큰(JWT) 등을 별도로 사용하여 클라이언트의 상태를 유지합니다.

3) 보안 취약성 (평문 전송)
문제: 데이터가 암호화되지 않은 **텍스트(평문)**로 전송됩니다. 해커가 중간에서 가로채면 로그인 정보나 개인정보가 그대로 노출됩니다.

해결: 이를 보완하기 위해 앞서 설명한 HTTPS를 사용합니다.
