3. HTTP의 특징과 문제점
**HTTP(HyperText Transfer Protocol)**는 웹에서 데이터를 주고받기 위한 가장 기본적인 통신 규약입니다.

핵심 특징
비연결성 (Connectionless): 클라이언트가 요청을 보내고 서버가 응답을 마치면 즉시 연결을 끊습니다. 서버 자원을 효율적으로 관리할 수 있게 해줍니다.

무상태성 (Stateless): 서버가 클라이언트의 이전 상태를 기억하지 않습니다. 따라서 각 요청은 독립적이며, 상태 유지를 위해 쿠키(Cookie)나 세션(Session)을 활용합니다.

문제점
평문 통신: 데이터를 암호화하지 않고 전송하므로, 중간에 패킷을 가로챌 경우 내용이 그대로 노출됩니다. (도청 위험)

변조 가능성: 통신 상에서 데이터가 수정되었는지 확인할 방법이 없습니다.

위장 가능성: 통신하는 상대방이 정말 의도한 서버인지 확인하는 절차가 부족합니다.
