1. 핵심 정의 (두괄식)
"JSON은 JavaScript Object Notation의 약자로, 데이터를 저장하고 전송하기 위한 경량의 텍스트 기반 데이터 교환 형식입니다."

2. 기술적 특징 및 장점
이 부분에서는 왜 JSON을 사용하는지 이유를 덧붙여 지식의 깊이를 보여줍니다.

가독성: "사람이 읽기 쉬운 텍스트 구조로 되어 있어 디버깅이 용이합니다."

효율성: "XML보다 구조가 단순하여 데이터 크기가 작고, 네트워크 전송 속도가 빠릅니다."

범용성: "특정 언어에 종속되지 않아 서버와 클라이언트, 혹은 서로 다른 언어로 작성된 시스템 간의 데이터 통신에 표준으로 사용됩니다."

3. 구조적 이해 (데이터 타입)
"기본적으로 Key-Value 쌍으로 이루어진 객체와 순서가 있는 리스트인 배열 구조를 가집니다. 숫자, 문자열, 불리언, 객체, 배열 등 다양한 타입을 중첩하여 복잡한 데이터 구조도 표현할 수 있습니다."
