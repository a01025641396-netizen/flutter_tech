1. 핵심 개념
네트워크 통신에서 데이터는 반드시 **문자열(String)**이나 바이트(Byte) 형태로 오가야 합니다. 하지만 프로그램 내부에서는 객체(Object/Class) 형태로 데이터를 다루죠. 이 간극을 메우는 과정이 직렬화와 역직렬화입니다.

직렬화 (Serialization):

객체 → 데이터 형식(JSON): Dart의 클래스 객체를 서버로 보내기 위해 JSON 문자열로 변환하는 과정입니다.

메서드 예시: toJson()

역직렬화 (Deserialization):

데이터 형식(JSON) → 객체: 서버로부터 받은 JSON 데이터를 다시 Dart 클래스 객체로 변환하여 코드에서 쓰기 쉽게 만드는 과정입니다.

메서드 예시: fromJson()

2. 왜 필요한가? (문제점과 해결)
문제: 서버가 보내주는 데이터는 단순히 {"name": "길동"} 같은 텍스트일 뿐입니다. 이를 코드에서 user["name"]처럼 일일이 호출하면 오타가 나기 쉽고 유지보수가 힘듭니다.

해결: 역직렬화를 통해 user.name처럼 **타입 안정성(Type Safety)**을 확보한 상태로 데이터를 다룰 수 있습니다.

3. Dart에서의 구현 방식
면접에서는 보통 두 가지 방식을 언급합니다.

수동 직렬화 (Manual Serialization):

클래스 안에 직접 fromJson과 toJson 메서드를 작성합니다.

규모가 작은 프로젝트에 적합하지만, 필드가 많아지면 실수할 확률이 높습니다.

자동 생성 방식 (Code Generation):

json_serializable, Freezed 같은 라이브러리를 사용합니다.

어노테이션(@JsonSerializable)을 붙이고 명령어를 실행하면 변환 코드가 자동으로 생성됩니다.

실무(대규모 프로젝트)에서 가장 선호하는 방식입니다.
