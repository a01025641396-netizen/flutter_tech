주요 역할 (Key Responsibilities)
① 데이터 가공 및 상태 관리 (State Management)
설명: 모델(Model)로부터 받은 원시 데이터를 뷰(View)가 바로 출력하기에 적합한 형태로 가공합니다.

포인트: 뷰는 '어떻게 보여줄지'에만 집중하게 하고, 뷰모델은 '무엇을 보여줄지'를 결정하는 상태를 보유합니다.

② 의존성 분리 (Decoupling)
설명: 뷰모델은 뷰에 대한 직접적인 참조를 가지지 않습니다.

포인트: 이는 UI 프레임워크로부터 비즈니스 로직을 독립시켜, UI 변경에 유연하게 대응할 수 있게 하며 코드의 재사용성을 높입니다.

③ 테스트 용이성 확보 (Testability)
설명: 뷰모델은 순수하게 데이터와 로직으로 구성되므로, UI 없이도 **단위 테스트(Unit Test)**가 가능합니다.

포인트: "UI 업데이트 로직이 비즈니스 로직과 분리되어 있어 검증이 용이하다"는 점을 강조하십시오.

④ 생명주기(Lifecycle)로부터의 독립성
설명: (특히 안드로이드와 같은 환경에서) 화면 회전 등 뷰가 파괴되고 재생성되는 상황에서도 데이터를 유지하여 사용자 경험을 보존합니다.
