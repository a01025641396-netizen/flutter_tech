# HashMap과 TreeMap의 차이점은 무엇인가요?

두 자료구조는 모두 **Key-Value Pair**를 저장하는 Map 인터페이스의 구현체이지만, 내부 구조와 성능 특징이 크게 다릅니다.

## 1. 핵심 비교

| 구분 | HashMap | TreeMap |
| :--- | :--- | :--- |
| **내부 구조** | **해시 테이블 (Hash Table)** | **이진 탐색 트리 (Red-Black Tree)** |
| **정렬 여부** | 순서 보장 안 됨 (해시값 기반) | **Key 기준 오름차순 정렬** 유지 |
| **시간 복잡도 (Get/Put)** | **O(1)** (평균) | **O(log N)** |
| **Null 허용** | Key에 Null 허용 (하나만) | Key에 Null 허용 안 됨 (비교 때문) |

## 2. HashMap (O(1))
- **특징:** 해시 함수를 통해 데이터의 위치를 결정하므로 매우 빠릅니다.
- **사용 상황:** 데이터의 정렬 순서가 중요하지 않고, 삽입/삭제/탐색 속도가 최우선인 경우에 사용합니다. (가장 일반적인 선택)

## 3. TreeMap (O(log N))
- **특징:** 모든 요소를 트리에 정렬된 상태로 유지합니다.
- **사용 상황:**
  - 데이터를 정렬된 순서로 순회해야 할 때.
  - 특정 값 범위 내의 데이터를 가져와야 할 때 (Range Search).
  - 전체에서 최솟값이나 최댓값을 효율적으로 찾아야 할 때.

## 4. 면접용 핵심 요약
"**HashMap**은 상수 시간 `O(1)`의 빠른 속도를 자랑하지만 순서를 보장하지 않으며, **TreeMap**은 이진 검색 트리를 활용하여 데이터를 정렬된 상태로 유지하므로 `O(log N)`의 속도를 가지지만 정렬과 범위 탐색에 유용합니다."

> [!WARNING]
> HashMap에서 해시 충돌(Hash Collision)이 잦아지면 성능이 최악의 경우 O(N)까지 저하될 수 있으나, 최신 Java 등의 언어에서는 이를 트리화하여 O(log N)으로 방어하기도 합니다.
