Provider: Flutter의 InheritedWidget을 기반으로 합니다. 따라서 위젯 트리에 의존하며, 데이터에 접근하기 위해서는 반드시 BuildContext가 필요합니다.

Riverpod: InheritedWidget을 사용하지 않고 완전히 새로 설계되었습니다. 위젯 트리에 의존하지 않는 컴파일 타임 안전성을 지닌 별도의 상태 관리 프레임워크입니다.

| 비교 항목 | Provider | Riverpod |
| :--- | :--- | :--- |
| **위젯 트리 의존성** | 필수 (트리 하단에서만 접근 가능) | 불필요 (전역 선언 및 자유로운 접근) |
| **BuildContext** | 접근 시 반드시 필요 | `WidgetRef` 또는 `ProviderRef`를 통해 접근 |
| **컴파일 안전성** | 런타임 에러 발생 가능성 높음 | 컴파일 타임에 대부분의 에러 포착 |
| **동일 타입 중복** | 동일한 타입의 Provider 다수 사용 곤란 | 변수명으로 구분되어 제약 없음 |
| **의존성 결합** | `ProxyProvider` 등으로 다소 복잡함 | 매우 직관적이고 선언적인 결합 가능 |
