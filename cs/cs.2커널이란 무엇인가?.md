1. 커널의 주요 기능
커널은 시스템의 하드웨어 자원을 효율적으로 배분하고 보호하기 위해 다음과 같은 4가지 핵심 기능을 수행합니다.

프로세스 관리: 여러 프로그램이 동시에 실행될 때, CPU를 언제 어느 만큼 사용할지 결정(Scheduling)하고 관리합니다.

메모리 관리: 각 프로세스가 독립된 메모리 공간을 사용할 수 있도록 스택과 힙을 할당하고, 서로의 영역을 침범하지 못하게 보호합니다.

장치 관리: 키보드, 마우스, 디스크 등 하드웨어 장치 드라이버를 통해 입출력을 제어합니다.

파일 시스템 관리: 데이터를 디스크에 저장하고 불러오는 방식과 권한을 관리합니다.

2. 시스템 콜 (System Call)
보안과 안정성을 위해 사용자 프로그램은 하드웨어에 직접 접근할 수 없습니다. 대신 커널에 **"대신 처리해달라"**고 요청을 보내는데, 이를 시스템 콜이라고 합니다.

유저 모드 (User Mode): 일반 애플리케이션이 실행되는 낮은 권한의 영역.

커널 모드 (Kernel Mode): 운영체제의 핵심 코드가 실행되는 높은 권한의 영역. 하드웨어 직접 제어가 가능함.

3. 커널의 종류
커널은 구조적 설계 방식에 따라 크게 두 가지로 분류됩니다.
단일형 커널 (Monolithic Kernel): 모든 기능을 하나의 거대한 커널 안에 넣음. 실행 속도가 빠르지만, 한 부분의 오류가 전체 시스템을 멈출 수 있음.

마이크로 커널 (Micro Kernel): 최소한의 기능만 커널에 넣고 나머지는 유저 영역으로 분리. 안정성이 높고 확장이 쉽지만, 서비스 간 통신 오버헤드로 인해 속도가 상대적으로 느림.
4. 요약
커널은 운영체제라는 정부의 **'실무 부서'**와 같습니다. 사용자가 내리는 명령을 하드웨어가 이해할 수 있는 전기 신호로 바꾸고, 자원이 낭비되지 않도록 통제하는 역할을 합니다.
