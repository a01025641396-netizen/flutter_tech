Gemini의 응답
컴퓨터는 0과 1만을 사용하는 이진수 체계를 사용하기 때문에, 우리가 일상에서 사용하는 십진수 소수점 데이터를 그대로 저장할 수 없습니다. 대신 **부동 소수점(Floating Point)**이라는 방식을 사용하여 실수를 표현합니다.

1. 부동 소수점(Floating Point) 방식
부동 소수점이란 소수점의 위치를 고정하지 않고, 마치 소수점이 "떠다니는(Floating)" 것처럼 표현하는 방식입니다. 현대의 거의 모든 컴퓨터는 IEEE 754라는 국제 표준 규격을 따릅니다.

실수는 다음과 같은 수식 형태로 메모리에 저장됩니다.

±(1.가수부)×2 
(지수부−바이어스)
 
2. 메모리 구조 (64비트 Double 기준)
컴퓨터는 소수점을 저장하기 위해 메모리 공간을 크게 세 부분으로 나눕니다.

부호 비트 (Sign bit): 1비트 사용. 0이면 양수, 1이면 음수를 나타냅니다.

지수부 (Exponent): 소수점의 위치를 결정합니다. 즉, 숫자의 크기(자릿수)를 나타냅니다.

가수부 (Fraction/Mantissa): 실제 숫자의 유효 숫자를 저장하는 부분입니다.

3. 부동 소수점의 특징과 한계
이러한 계산 방식 때문에 프로그래밍 시 반드시 주의해야 할 점들이 있습니다.

부동 소수점 오차: 십진수 0.1은 이진수로 변환하면 $0.00011001100...$과 같이 끝이 없는 무한 소수가 됩니다. 메모리는 한정되어 있으므로 뒷부분을 잘라내는데, 이 과정에서 미세한 **반올림 오차(Rounding Error)**가 발생합니다.

비교 연산의 위험: 위와 같은 오차 때문에 0.1 + 0.2 == 0.3을 계산하면 결과가 false로 나올 수 있습니다. 실제 계산 값은 0.30000000000000004와 같이 미세하게 다르기 때문입니다.

고정 소수점과의 차이: 소수점 위치를 고정하는 '고정 소수점' 방식보다 훨씬 넓은 범위의 숫자를 표현할 수 있어 범용적으로 사용됩니다.

4. 요약
복사하여 활용하시기 편하도록 정리해 드립니다.

부동 소수점 (Floating Point): IEEE 754 표준에 따라 실수를 부호, 지수, 가수 세 부분으로 나누어 이진수로 저장하는 방식.

장점: 아주 작은 수부터 매우 큰 수까지 넓은 범위를 효율적으로 표현 가능함.

단점: 십진수를 이진수로 변환하는 과정에서 미세한 수치 오차가 발생할 수 있음.

해결책: 금융권처럼 정확한 계산이 필수인 경우 Decimal 타입을 사용하거나, 정수로 변환하여 계산한 뒤 다시 소수점으로 나누는 방식을 사용함.
