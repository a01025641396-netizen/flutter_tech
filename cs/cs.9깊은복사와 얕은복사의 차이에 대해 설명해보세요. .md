데이터를 복사할 때 참조값(주소)만 복사하느냐, 실제 값 자체를 새로 생성하느냐에 따라 **얕은 복사(Shallow Copy)**와 **깊은 복사(Deep Copy)**로 나뉩니다.

1. 얕은 복사 (Shallow Copy)
객체를 복사할 때, 해당 객체 내의 참조값(메모리 주소)만 복사하는 방식입니다.

특징: 복사본을 수정하면 원본 데이터도 함께 변경됩니다. 두 객체가 같은 메모리 주소를 가리키고 있기 때문입니다.

성능: 실제 데이터를 새로 생성하지 않으므로 속도가 빠르고 메모리 사용량이 적습니다.

예시: List B = A; 와 같이 대입 연산자를 사용하는 경우.

2. 깊은 복사 (Deep Copy)
객체를 복사할 때, 실제 값들을 모두 복사하여 새로운 메모리 공간에 할당하는 방식입니다.

특징: 원본과 복사본이 완전히 독립적입니다. 복사본을 수정해도 원본에 아무런 영향을 주지 않습니다.

성능: 새로운 데이터를 위한 메모리를 할당하고 내용을 채워야 하므로 얕은 복사보다 느리고 메모리를 더 많이 사용합니다.

예시: 객체의 모든 필드를 순회하며 새 객체에 값을 일일이 할당하거나, 직렬화(Serialization)를 이용하는 경우.

3. 핵심 차이점 비교 
얕은 복사 (Shallow Copy): 주소값만 복사. 원본과 복사본이 동일한 자원을 공유함. 한쪽을 수정하면 다른 쪽도 영향을 받음.

깊은 복사 (Deep Copy): 실제 값 자체를 새로운 메모리에 복사. 원본과 복사본이 독립적인 자원을 가짐. 서로 영향을 주지 않음.
