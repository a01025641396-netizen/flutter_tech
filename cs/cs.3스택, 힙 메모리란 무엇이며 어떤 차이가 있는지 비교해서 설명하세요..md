1. 스택 (Stack)
스택은 함수 호출과 관련된 지역 변수와 매개변수가 저장되는 정적 메모리 영역입니다. 자료구조의 스택과 마찬가지로 후입선출(LIFO, Last In First Out) 방식으로 동작합니다.

특징:

함수가 호출될 때 할당되고, 함수가 종료되면 자동으로 제거됩니다.

컴파일 타임에 크기가 결정되며, 메모리 할당 속도가 매우 빠릅니다.

공간이 한정되어 있어 너무 큰 데이터를 넣거나 재귀 호출이 깊어지면 Stack Overflow가 발생합니다.

저장 데이터: 기본값 형식(int, double, bool 등), 함수 호출 프레임, 지역 변수.

2. 힙 (Heap)
힙은 사용자가 직접 관리하거나 가비지 컬렉터가 관리하는 동적 메모리 영역입니다. 프로그램 실행 중(Runtime)에 크기가 결정됩니다.

특징:

메모리 크기에 제한이 거의 없으며, 전역적으로 접근이 가능합니다.

메모리를 할당하고 해제하는 과정이 필요하며, 스택에 비해 속도가 느립니다.

가비지 컬렉터(GC)가 주기적으로 불필요한 데이터를 정리해야 성능 저하를 막을 수 있습니다.

저장 데이터: 참조 형식(클래스 인스턴스, 배열, 객체 등).

3. 스택 vs 힙 비교 요약
복사하여 활용하기 편하도록 리스트 형태로 정리해 드립니다.

스택 (Stack): 함수의 생명주기와 함께하며 자동으로 관리됨. 할당 속도가 매우 빠르나 공간이 작음. (정적 할당)

힙 (Heap): 런타임에 크기가 결정되며 가비지 컬렉터가 관리함. 할당 속도는 상대적으로 느리나 큰 데이터를 저장할 수 있음. (동적 할당)

상호작용: 참조 형식의 경우, 실제 데이터는 힙에 저장되고 그 데이터가 위치한 메모리 주소(참조값)는 스택에 저장됩니다.
