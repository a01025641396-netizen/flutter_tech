객체 지향 프로그래밍에서 추상화란 복잡한 내부 구현 로직을 숨기고, 사용자에게 필요한 **핵심적인 기능(인터페이스)**만을 추출하여 정의하는 것을 의미합니다.

쉽게 말해, '무엇을 하는지'는 정의하지만 '어떻게 하는지'는 세부적으로 드러내지 않는 설계 기법입니다.

추상화를 사용하는 이유
복잡성 감소: 사용자는 내부의 복잡한 코드를 몰라도 클래스가 제공하는 기능만 알면 즉시 사용할 수 있습니다.

표준화(가이드라인): 여러 개발자가 협업할 때, 반드시 구현해야 하는 메서드 이름을 통일시켜 실수를 방지합니다. (예: 모든 가전제품은 turnOn이라는 이름의 함수를 가져야 함)

유연성 및 유지보수: 내부 로직을 수정하더라도 외부로 노출된 인터페이스(함수 이름 등)가 바뀌지 않는다면, 해당 기능을 사용하는 다른 코드들을 수정할 필요가 없습니다.

플러터(Dart)에서는 abstract 키워드를 사용하여 추상 클래스를 만듭니다. 이 클래스는 직접 객체(Instance)를 생성할 수 없으며, 오로지 다른 클래스의 설계도 역할만 수행합니다.
