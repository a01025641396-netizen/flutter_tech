OOP는 Object-Oriented Programming의 약자로, 우리말로 **'객체지향 프로그래밍'**이라고 부릅니다.

앞서 설명드린 '객체지향' 그 자체를 의미하며, 프로그램을 작성하는 하나의 방법론이자 패러다임입니다. 현대 소프트웨어 개발(Flutter, Java, Python 등)에서 가장 널리 쓰이는 방식입니다.

1. OOP의 핵심 요약
프로그램을 "순차적으로 실행되는 명령어의 집합"으로 보지 않고, **"상태와 기능을 가진 '객체'들의 모임"**으로 보는 것입니다.

객체(Object): 데이터(변수)와 그 데이터를 처리하는 동작(함수)을 하나로 묶은 단위입니다.

지향(Oriented): 그것을 중심으로 설계하고 개발한다는 뜻입니다.

2. OOP를 구성하는 3단계 과정
설계 (Class): 현실 세계의 사물을 분석하여 '설계도'를 만듭니다. (예: 자동차 설계도)

생성 (Instance): 설계도를 바탕으로 메모리에 실제 '객체'를 만듭니다. (예: 실제 아반떼, 테슬라 차량)

소통 (Message): 만들어진 객체들이 서로 데이터를 주고받으며 기능을 수행합니다. (예: 운전자가 가속 페달 객체를 밟으면 엔진 객체가 작동함)



3. OOP의 반대 개념: 절차지향 (Procedural Programming)
절차지향 (C언어 등): 실행 순서가 중요합니다. 요리 레시피처럼 1번 하고 2번 하는 방식입니다. 프로그램이 커지면 코드가 꼬여서 수정하기 매우 어렵습니다.

OOP (Dart, Java 등): 부품 중심입니다. 고장 난 부품(객체)만 갈아 끼우면 되므로 대규모 시스템 유지보수에 압도적으로 유리합니다.
