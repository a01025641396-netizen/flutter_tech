# 람다식(Lambda Expression)이란 무엇인가요?

## 1. 정의
**람다식(Lambda Expression)**은 함수를 하나의 **간결한 식(Expression)**으로 표현한 것입니다. 근본적으로는 **익명 함수(Anonymous Function)**의 한 형태이며, 함수를 값처럼 다룰 수 있게 하여 변수에 할당하거나 인자로 전달할 수 있습니다.

## 2. 특징
- **간결성:** 불필요한 메서드 선언이나 클래스 정의 없이 함수의 핵심 로직만 구현합니다.
- **일급 객체화:** 함수 자체가 데이터처럼 취급되어 고차 함수(High-order function)의 인자로 넘기기 최적화되어 있습니다.
- **가독성:** `filter`, `map`, `reduce` 등의 연산과 결합하여 코드를 선언적으로 만듭니다.

## 3. 익명 함수 vs 익명 클래스 (Java 기준)
- **익명 클래스:** 클래스 정의와 인스턴스 생성을 동시에 하며, 장황한 코드가 필요합니다.
- **람다식:** 인터페이스에 메서드가 하나만 있는 '함수형 인터페이스'를 타겟으로 하여, 매우 압축된 문법을 제공합니다.

## 4. 활용 예시 (Dart)
```dart
// 일반적인 함수
int add(int a, int b) {
  return a + b;
}

// 람다식 (화살표 함수)
var addLambda = (int a, int b) => a + b;

// List 연산 활용
var numbers = [1, 2, 3];
var doubled = numbers.map((n) => n * 2).toList();
```

> [!NOTE]
> 람다식은 객체지향 패러다임에 **함수형 프로그래밍**의 요소를 결합하여 현대적인 프로그래밍 언어에서 코드를 훨씬 더 유연하고 간결하게 만들어 줍니다.
