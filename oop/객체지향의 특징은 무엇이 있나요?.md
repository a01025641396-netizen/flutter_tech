객체지향 프로그래밍(OOP)의 핵심은 현실 세계의 사물이나 개념을 소프트웨어 안의 **'객체'**로 모델링하는 것입니다. 이를 효과적으로 구현하기 위한 4가지 핵심 특징은 다음과 같습니다.

1. 캡슐화 (Encapsulation)
객체의 상태(데이터)와 행위(메서드)를 하나로 묶고, 실제 구현 내용 일부를 외부에 감추는 것을 의미합니다.

정보 은닉: 외부에서 객체 내부의 데이터를 직접 수정하지 못하게 막고, 정해진 메서드를 통해서만 접근하게 합니다.

장점: 내부 코드를 수정해도 외부 코드에 영향을 주지 않아 유지보수가 쉬워지며, 데이터의 무결성을 보호할 수 있습니다.

2. 상속 (Inheritance)
이미 정의된 부모 클래스의 모든 특성을 자식 클래스가 물려받는 것입니다.

재사용성: 공통된 기능을 부모 클래스에 한 번만 작성하면, 여러 자식 클래스에서 그대로 사용할 수 있습니다.

확장: 부모의 기능을 그대로 쓰면서도 자식 클래스만의 새로운 기능을 추가할 수 있습니다.

3. 추상화 (Abstraction)
객체의 복잡한 내부 로직을 숨기고, 사용자에게 필요한 **핵심적인 인터페이스(기능)**만을 추출하여 정의하는 과정입니다.

단순화: "무엇을 하는가"에 집중하고 "어떻게 구현하는가"는 뒤로 미룹니다.

설계도 역할: 추상 클래스를 통해 공통적인 규격을 만들어 개발자 간의 약속을 정의합니다.

4. 다형성 (Polymorphism)
하나의 인터페이스나 메서드가 상황에 따라 서로 다른 방식으로 동작하는 성질입니다.

오버라이딩(Overriding): 부모 클래스의 메서드를 자식 클래스에서 자기 방식대로 재정의하여 사용하는 것입니다.

유연성: 동일한 명령(예: "소리를 내라")을 내려도, 객체가 '개'라면 "멍멍", '고양이'라면 "야옹"하고 각자 다르게 반응하게 할 수 있습니다.
