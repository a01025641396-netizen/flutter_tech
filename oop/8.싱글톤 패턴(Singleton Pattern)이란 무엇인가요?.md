# 싱글톤 패턴(Singleton Pattern)이란 무엇인가요?

## 1. 정의
**싱글톤 패턴(Singleton Pattern)**은 애플리케이션 전체에서 특정 클래스의 **인스턴스가 단 하나만 생성**되도록 보장하고, 어디서든 그 인스턴스에 접근할 수 있는 전역적인 접근점을 제공하는 디자인 패턴입니다.

## 2. 왜 사용하는가? (Pros)
- **자원 절약:** 데이터베이스 연결 풀, 설정 관리자, 로그 기록기 등 여러 번 생성할 필요가 없는 객체의 메모리 낭비를 줄입니다.
- **데이터 공유:** 여러 컴포넌트 간에 공통된 상태를 공유하기가 매우 편리합니다.

## 3. 단점 및 주의사항 (Cons)
- **Tightly Coupled:** 전역적으로 사용되므로 객체 간 결합도가 높아져 단위 테스트(Unit Test)가 어려워질 수 있습니다.
- **멀티스레드 이슈:** 인스턴스 생성 시점에 여러 스레드가 동시에 접근하면 중복 생성될 위험이 있으므로 `Critical Section` 처리가 필요합니다.
- **상태 관리:** 싱글톤이 너무 많은 상태를 가지면 'God Object'가 되어 유지보수가 힘들어집니다.

## 4. 구현 예시 (Dart)
```dart
class DatabaseClient {
  // 1. 내부적인 정적 인스턴스 생성
  static final DatabaseClient _instance = DatabaseClient._internal();

  // 2. 외부 호출용 Factory 생성자
  factory DatabaseClient() => _instance;

  // 3. 내부 전용 생성자 (Private)
  DatabaseClient._internal();
}
```

> [!IMPORTANT]
> 현대적인 프레임워크(Flutter의 Riverpod, GetIt 등)에서는 싱글톤을 직접 구현하기보다는 **의존성 주입(Dependency Injection)** 컨테이너를 통해 인스턴스의 생명주기를 관리하는 방식을 더 권장합니다.
