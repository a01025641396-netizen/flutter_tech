1. 위젯 트리의 핵심 개념
계층적 구조: 최상단에는 MaterialApp이나 CupertinoApp 같은 루트 위젯이 위치하며, 그 아래로 Scaffold, Center, Column, Text 등이 자식(Child) 또는 자식들(Children)로 연결됩니다.

선언적 UI: 플러터는 "상태가 바뀌면 UI가 바뀐다"는 선언적 방식을 사용합니다. 위젯 트리는 현재 상태에서 UI가 어떻게 보여야 하는지를 설명하는 설계도와 같습니다.

불변성(Immutability): 위젯 객체 자체는 불변입니다. 상태가 변경되어 화면을 다시 그려야 할 때, 플러터는 기존 트리를 수정하는 대신 새로운 위젯 트리를 빠르게 생성합니다.

2. 플러터의 3가지 트리 (Three Trees)
플러터는 성능 최적화를 위해 위젯 트리 외에도 내부적으로 두 개의 트리를 더 관리합니다.

Widget Tree (설계도): 개발자가 코드로 작성하는 구성 요소입니다. 매우 가볍고 자주 교체됩니다.

Element Tree (연결고리): 위젯 트리와 렌더링 트리를 연결하는 중재자입니다. 위젯의 위치(BuildContext)를 관리하며, 상태(State)가 저장되는 곳입니다.

Render Object Tree (실제 화면 출력): 화면의 크기, 위치, 레이아웃을 계산하고 실제로 픽셀을 그리는 무거운 객체들의 트리입니다.
