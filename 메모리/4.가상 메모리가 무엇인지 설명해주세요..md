**가상 메모리(Virtual Memory)**란 실제 컴퓨터에 설치된 물리적 RAM의 용량보다 더 큰 프로그램이나 여러 프로그램을 동시에 실행할 수 있도록 해주는 메모리 관리 기법입니다.

애플리케이션이 물리적 메모리의 실제 주소를 직접 다루는 것이 아니라, OS가 제공하는 가상 주소 공간을 사용하여 마치 메모리를 독점하고 있는 것처럼 착각하게 만드는 기술입니다.

1. 가상 메모리의 핵심 원리
가상 메모리의 핵심은 **"프로그램 전체가 메모리에 올라와 있지 않아도 실행이 가능하다"**는 점입니다.

요구 페이징 (Demand Paging): 프로그램 실행 시 당장 필요한 부분만 실제 RAM에 올리고, 나머지는 보조 기억장치(SSD/HDD)의 **스왑 영역(Swap Area)**에 저장해둡니다.

매핑 (Mapping): CPU가 가상 주소를 요청하면, OS와 하드웨어(MMU)가 협력하여 이를 실제 RAM의 물리 주소로 번역하여 데이터를 찾아옵니다.

2. 가상 메모리의 주요 구성 요소
가상 주소 (Virtual Address): 프로세스가 참조하는 주소입니다. 모든 프로세스는 0번지부터 시작하는 독립적인 가상 공간을 가집니다.

물리 주소 (Physical Address): 실제 RAM 부품에 할당된 주소입니다.

MMU (Memory Management Unit): 가상 주소를 물리 주소로 변환해주는 전용 하드웨어 장치입니다.

페이지 테이블 (Page Table): 가상 주소와 물리 주소의 연결 정보를 담고 있는 지도 역할을 합니다.

3. 가상 메모리를 사용하는 이유 (장점)
메모리 용량 한계 극복: 8GB RAM을 가진 컴퓨터에서도 16GB가 넘는 대형 게임이나 프로그램을 실행할 수 있습니다.

메모리 보호: 각 프로세스는 자신만의 가상 주소 공간을 가지므로, 다른 프로세스의 메모리 영역을 훔쳐보거나 수정할 수 없어 보안과 안정성이 높습니다.

메모리 공유: 여러 프로세스가 공통으로 사용하는 라이브러리(예: printf 함수 등)를 각각 메모리에 올리지 않고, 물리 메모리 상의 한 곳만 가상 주소로 연결하여 효율적으로 공유할 수 있습니다.

4. 발생할 수 있는 문제점
페이지 폴트 (Page Fault): 가상 주소로 접근했는데 실제 RAM에 해당 데이터가 없을 때 발생합니다. 이때 OS는 보조 기억장치에서 데이터를 가져와야 하므로 시간이 걸립니다.

스래싱 (Thrashing): RAM 용량이 너무 부족하여 데이터 교체(Page Swap)가 너무 빈번하게 일어나, 실제 연산보다 데이터를 옮기는 데 CPU 시간을 다 써버려 시스템이 매우 느려지는 현상입니다.
