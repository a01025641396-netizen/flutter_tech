Dart는 특히 **Flutter 프레임워크의 UI 렌더링(60fps/120fps)**을 부드럽게 유지하기 위해 매우 효율적이고 빠른 가비지 컬렉션(GC) 방식을 채택하고 있습니다. Dart의 GC는 크게 세대별 수집(Generational Collection) 전략을 사용하며, 두 개의 층으로 나누어 작동합니다.

1. 세대별 가비지 컬렉션 (Generational GC)
Dart는 객체의 생명 주기에 따라 메모리를 **Young Space(New Space)**와 Old Space로 나누어 관리합니다.

① Young Generation (Young Space)
대부분의 객체는 생성된 후 아주 짧은 시간만 사용되고 버려집니다(예: Stateless 위젯, 일회성 데이터 변수).

작동 방식: 'Scavenging'이라는 기법을 사용합니다. 메모리를 두 개의 세그먼트로 나누어, 사용 중인(Live) 객체만 다른 쪽으로 복사하고 기존 공간을 통째로 비워버립니다.

특징: 복사하는 방식이라 매우 빠르며, UI 렌더링에 영향을 거의 주지 않을 만큼 가볍습니다.

② Old Generation (Old Space)
Young Space에서 여러 번의 GC를 거치고도 살아남은 객체들이 옮겨오는 공간입니다.

작동 방식: 'Mark-Sweep-Compact' 알고리즘을 사용합니다.

Mark: 사용 중인 객체에 표시를 합니다.

Sweep: 표시되지 않은 가비지 객체를 제거합니다.

Compact: 파편화된 메모리 공간을 하나로 모아 정리합니다.

특징: Young Space보다 검사 주기가 길지만, 작업량이 많아 중단 시간(Stop-the-world)이 발생할 가능성이 있습니다.

2. Flutter에 최적화된 Dart GC의 특징
Dart GC가 다른 언어와 차별화되는 점은 Flutter의 UI 스케줄링과 긴밀하게 협력한다는 것입니다.

초당 60회 이상의 렌더링 대응: Flutter는 1초에 60번 이상 화면을 다시 그립니다. Dart GC는 화면이 그려지지 않는 **유휴 시간(Idle Time)**을 감지하여 그 사이에 GC 작업을 수행하려고 노력합니다. 덕분에 사용자는 화면 끊김(Jank)을 거의 느끼지 못합니다.

불변 객체(Immutable Objects) 최적화: Flutter 위젯은 대부분 불변 객체입니다. Dart는 이러한 불변 객체들을 빠르게 생성하고 회수하는 데 최적화되어 있어, 수많은 위젯이 매 프레임 생성되어도 성능 저하가 적습니다.
