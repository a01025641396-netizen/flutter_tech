운영체제(OS)가 메모리를 관리하는 것은 한정된 땅(RAM) 위에 수많은 건물(프로세스)을 효율적이고 안전하게 짓는 도시 계획과 같습니다.

1. OS가 메모리 관리를 해야 하는 이유
만약 OS가 메모리를 관리하지 않는다면, 개발자가 직접 빈 메모리 주소를 찾아 데이터를 저장해야 하며 다음과 같은 심각한 문제가 발생합니다.

한정된 자원의 효율적 배분: RAM 용량은 실행하려는 모든 프로그램의 크기보다 작을 때가 많습니다. 이를 쪼개서 효율적으로 써야 합니다.

격리 및 보호: 특정 프로그램이 실수나 악의적인 의도로 다른 프로그램의 메모리 영역을 침범하거나 OS의 핵심 영역을 건드리는 것을 방지해야 합니다.

편의성 (추상화): 개발자가 실제 물리적인 RAM 주소를 일일이 계산하지 않고도, 마치 메모리를 독점하고 있는 것처럼 느끼게 해주는 가상 메모리 환경을 제공해야 합니다.

2. 메모리 관리 전략
OS는 메모리를 효율적으로 사용하기 위해 크게 3가지 단계의 전략을 사용합니다.

① 반입 전략 (Fetch Strategy)
'언제' 데이터를 메모리로 가져올 것인가에 대한 전략입니다.

요구 반입 (Demand Fetch): 실행 중인 프로그램이 특정 데이터를 필요로 할 때 메모리에 올립니다.

예상 반입 (Anticipatory Fetch): 참조될 가능성이 높은 데이터를 미리 메모리에 올려두는 방식입니다.

② 배치 전략 (Placement Strategy)
'어디에' 데이터를 저장할 것인가에 대한 전략입니다.

최초 적합 (First Fit): 가용 공간 중 가장 먼저 발견된 빈 곳에 배치합니다. (속도 빠름)

최적 적합 (Best Fit): 빈 공간 중 데이터 크기와 가장 차이가 적은 곳에 배치합니다. (공간 효율적)

최악 적합 (Worst Fit): 가장 큰 빈 공간에 배치합니다. (남은 공간을 다시 활용하기 좋음)

③ 교체 전략 (Replacement Strategy)
메모리가 꽉 찼을 때 '어떤 것'을 내보낼 것인가에 대한 전략입니다.

FIFO (First-In-First-Out): 가장 먼저 들어온 페이지를 먼저 내보냅니다.

LRU (Least Recently Used): 가장 오랫동안 사용되지 않은 페이지를 내보냅니다. (성능 우수)

LFU (Least Frequently Used): 참조 횟수가 가장 적은 페이지를 내보냅니다.

3. 핵심 기술: 가상 메모리 (Virtual Memory)
현대 OS 메모리 관리의 꽃은 가상 메모리입니다.

페이징 (Paging): 물리적 메모리를 고정된 크기(Page)로 나누어 관리하는 방식입니다. 외부 단편화 문제를 해결합니다.

세그먼테이션 (Segmentation): 논리적인 단위(Code, Data, Stack 등)로 나누어 관리하는 방식입니다. 보호와 공유에 유리합니다.
