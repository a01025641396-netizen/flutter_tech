데이터 영역(Data Segment)에 함께 저장되는 **전역 변수(Global Variable)**와 **정적 변수(Static Variable)**는 생성 시점과 소멸 시점은 동일하지만, **접근할 수 있는 범위(Scope)**와 가시성에서 결정적인 차이가 있습니다.

1. 전역 변수 (Global Variable)
프로그램 전체에서 공유되는 변수입니다.

선언 위치: 함수 외부(파일의 최상단)에 선언됩니다.

접근 범위: 프로그램 내의 모든 소스 파일에서 접근할 수 있습니다. (외부 파일에서 extern 키워드 등을 사용해 접근 가능)

장점: 여러 함수나 파일에서 공통으로 데이터를 관리할 때 편리합니다.

단점: 어디서든 값을 바꿀 수 있어 데이터의 출처를 파악하기 어렵고, 의도치 않은 수정(사이드 이펙트)이 발생할 위험이 큽니다.

2. 정적 변수 (Static Variable)
특정 범위(Scope) 내에서만 사용되지만, 프로그램이 종료될 때까지 메모리가 유지되는 변수입니다.

선언 위치: 함수 내부 또는 특정 소스 파일 내부에 static 키워드와 함께 선언됩니다.

접근 범위: * 함수 내부 static: 해당 함수 안에서만 보입니다. 하지만 함수가 끝나도 값은 사라지지 않고 유지됩니다.

파일 내부 static: 해당 소스 파일 안에서만 접근 가능하며, 다른 파일에서는 보이지 않습니다.

장점: 데이터의 생명 주기는 전역 변수처럼 유지하면서도, 접근 권한을 제한하여 캡슐화와 보안성을 높일 수 있습니다.
