메모리(RAM)가 고갈되면 운영체제는 시스템 붕괴를 막기 위해 가용한 모든 수단을 동원하며, 이 과정에서 사용자는 급격한 성능 저하와 실행 중단 현상을 경험하게 됩니다. 주요 단계별 현상은 다음과 같습니다.

1. 스와핑(Swapping)과 스래싱(Thrashing) 발생
RAM 공간이 부족해지면 OS는 당장 사용하지 않는 메모리 데이터를 보조 기억장치(SSD/HDD)의 스왑 영역으로 옮깁니다.

성능 저하: SSD는 RAM보다 수십 배 이상 느리기 때문에, 데이터를 옮기고 다시 가져오는 과정에서 시스템이 눈에 띄게 느려집니다.

스래싱 현상: 메모리 부족이 심해지면 CPU가 실제 계산 업무보다 메모리 페이지를 디스크로 옮기고 가져오는(Page In/Out) 작업에 더 많은 시간을 쓰게 됩니다. 이때 마우스 커서가 멈추거나 창 전환이 안 되는 등 컴퓨터가 사실상 마비되는 상태가 됩니다.

2. 메모리 할당 실패 및 앱 강제 종료
프로그램이 새로운 객체를 생성하거나 데이터를 저장하려고 할 때, OS가 메모리 할당 요청을 거부하게 됩니다.

Out of Memory (OOM) 에러: 프로그램 내부에서 메모리 할당 실패 예외가 발생합니다.

크래시(Crash): 예외 처리가 되어 있지 않은 앱은 그 즉시 "응답 없음" 상태가 되거나 갑자기 꺼져버립니다.

3. OOM 킬러(OOM Killer)의 동작 (Linux/Android 등)
OS 차원에서 시스템 전체가 뻗는 것을 방지하기 위해 가동하는 '최후의 수단'입니다.

강제 종료: 운영체제는 현재 실행 중인 프로세스들 중 우선순위가 낮거나 메모리를 지나치게 많이 점유하는 프로세스를 강제로 종료시켜 메모리를 강제 회수합니다.

현상: 멀티태스킹 중에 배경에 있던 앱이 사라지거나, 웹 브라우저의 탭이 새로고침되는 현상이 이에 해당합니다.

4. 시스템 패닉 및 블루스크린 (Kernel Panic / BSOD)
OS의 핵심 영역인 커널조차 메모리를 할당받지 못하는 상황에 이르면 시스템은 더 이상 동작할 수 없습니다.

전체 중단: 시스템 보호를 위해 모든 동작을 멈추고 에러 화면(블루스크린 등)을 띄우며 재부팅을 유도합니다.
