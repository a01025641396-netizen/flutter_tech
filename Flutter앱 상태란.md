# 1.Flutter에서 상태란 앱이 실행되는 동안 변할 수 있고 그 변화가 사용자 인터페이스에 
# 즉각적으로 영향을 미치는 모든 데이터를 의미합니다.
# 요약하면 Flutter에서 상태란 화면의 그려질 데이터 그 자체를 의미하며, 이 상태를
# 어떻게 구조화하고 변화시키느냐가 효율적인 앱 개발의 핵심입니다.

# 2. 상태의 두 가지 유형
# Flutter에서는 상태가 미치는 범위에 따라 크게 두 가지로 분류합니다.

# 단기 상태 (Ephemeral State): 특정 위젯 내에서만 유효한 로컬 상태입니다. 다른 위젯과 공유할 필요가 없는 데이터를 관리할 때 사용합니다.

 # 예시: 현재 선택된 탭 번호, 텍스트 필드에 입력 중인 문자열, 애니메이션의 현재 진행 상황 등.

# 앱 상태 (App State): 앱의 여러 부분에서 공유되거나 사용자 세션 동안 유지되어야 하는 상태입니다.

# 예시: 로그인 정보, 사용자 환경 설정, 장바구니에 담긴 상품 목록, 알림 데이터 등.
